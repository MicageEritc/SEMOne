# 2026-06-26

## SEMHub 项目初始化

创建了全新的 SEMHub 项目 —— 扫描电子显微镜知识平台。

### 技术栈
- Next.js 16.2.9 (Turbopack)
- React 19.2.4
- TypeScript 5
- Tailwind CSS 4
- App Router

### 核心功能
- **Markdown 文章系统**：`articles/*.md` 自动生成页面，支持 frontmatter 元数据
- **文章分类**：guide(开始学习)、knowledge(知识库)、case(案例中心)、tool(工具)
- **动态路由**：`/articles/[slug]` 使用 `generateStaticParams` 静态生成
- **Markdown 渲染**：gray-matter + unified + remark-parse + remark-html

### 设计风格
- Apple 风格极简设计，白色+深灰+少量蓝(#0071e3)
- 无渐变，无科技蓝，无暗色模式
- 响应式布局，移动端适配

### 文件结构
- `articles/` — Markdown 文章目录
- `src/app/` — Next.js App Router 页面
- `src/components/` — 可复用组件(SearchBox, EntryCard, Header, Footer)
- `src/lib/` — 工具函数(articles.ts)
- `src/types/` — TypeScript 类型定义

## 最终架构重构

### 删除的旧逻辑
- `src/data/article-index.ts` — 完全删除，不再手动维护文章索引
- `article.meta.category` — 不再手写，由文件夹路径自动推导
- `article.meta.published` — 改为 `draft`（默认已发布）
- Knowledge Tree 中的文章数据 — 只保留栏目结构

### Content Engine 自动扫描流程
1. 扫描 `articles/` 下所有分区目录（guide/knowledge/cases/...）
2. 递归查找所有 `.md` 文件（排除 README.md）
3. 解析 frontmatter，跳过 `draft: true` 的文章
4. 自动推导 section（一级目录名）、category（直接父目录名）、slug（文件路径）
5. 按 order + date 排序，生成所有列表和导航
